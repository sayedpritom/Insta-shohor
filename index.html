How Does Javascript Work?

Javascript is a client-side programming language. That means it runs on the browsers like chrome, firefox, etc. To run in browsers javascript uses certain engines. For example, V8 for chrome, spider monkey for firefox, Chakra for Microsoft Edge & the list goes on.

Javascript is an interpreted language. Which means it can be executed directly without the source code being compiled in the first place.

Step 1: In the first place, a parser receives the code & checks line by line if there is any syntactical error. If no error is found then goes to the next step.

Step 2: In phase two, an Abstract Syntex Tree is created. In short, it's called AST.
Step 3: In phase three, the Abstract Syntax Tree is converted into machine code.

Step 4: Finally the machine code is ready & executed by the main computer.

Difference Between Local Storage & Session Storage

Local storage and session storage are almost identical except for one thing. That is, data stored in session storage will be deleted after the window is closed & only persists for the duration of the browser session. But the data will not be cleaned if the page is reloaded. On the other hand, local storage saves the data for later usage & the data is not cleaned even after the browser tab is closed.
